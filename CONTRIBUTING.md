# Contributing to Awesome Prometheus

Additions and corrections are welcome.

## Adding an entry

- The link must point to something useful for working with Prometheus in practice.
- Write a short, factual description in one sentence. Say what it does, not that it is "great" or "powerful".
- Place the entry under the most relevant section.
- Check that the link is live before submitting.
- Run `npx awesome-lint` and fix any failures before opening a PR.

## Updating an entry

If a description is outdated or a link points to the wrong page, open a PR with the corrected version.

## Removing an entry

If a resource is no longer maintained or a link has been dead for more than 30 days, open an issue or PR to remove it.

## Pull request process

1. Fork the repository.
2. Make your changes in a new branch.
3. Open a PR with a short description of what you added and why it belongs here.

Entries with no clear connection to Prometheus or that duplicate existing entries will be declined.
